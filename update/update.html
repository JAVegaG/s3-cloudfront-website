Introduction

This Dashboard was designed as a Graphic User Interface to implement CRUD operations over an AWS DynamoDB Table. The DynamoDB Table itself is accessed through Lambda functions that are triggered by an API Gateway petition. The table name, the URL to directly interact with the API, and the explanation of how the infrastructure was implemented can be found in JAVegaG/terraform-aws. On the other hand, the explanation of how the code of the lambdas is updated can be found in JAVegaG/aws-lambdas.

Operations

As stated above, this GUI is supposed to work as a control panel for the main and basic operations of persistent storage, i.e. create a new record, read all records, read, update and delete a specific record based on a primary key, provided to the operation. Notice that each link on the menu bar is one of the previously mentioned procedures, so to get started just go ahead and press the one that you would like to try.
